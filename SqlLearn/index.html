SQL Fillestare
Konceptet bazë — SELECT, WHERE, ORDER BY, dhe të tjera.
01
SELECT — Merr të dhënat
Komanda bazë për të lexuar të dhëna nga tabela
Fillestare

SELECT është komanda e parë që çdo programues SQL mëson. Ajo "zgjedh" kolona nga tabela.

-- Merr të gjitha kolonat nga tabela SELECT * FROM punonjësit; -- Merr vetëm kolonat specifike SELECT emri, mbiemri, rroga FROM punonjësit;
1
SELECT * — ylli (*) do të thotë "të gjitha kolonat". Shpejt por jo eficiente për tabela të mëdha.
2
SELECT emri, rroga — specifiko kolonat që dëshiron. Kjo është praktika e mirë.
3
FROM tabela — tregon nga cila tabelë merren të dhënat. Gjithmonë pas SELECT.
02
WHERE — Filtro të dhënat
Kthehu vetëm rreshtat që plotësojnë kushtin
Fillestare

WHERE filtron rreshtat. Vetëm rreshtat që plotësojnë kushtin kthehen.

SELECT emri, rroga FROM punonjësit WHERE rroga > 50000; -- Tekst me thonjëza SELECT * FROM punonjësit WHERE departamenti = 'IT'; -- Kombinim me AND / OR WHERE rroga > 40000 AND departamenti = 'HR';
●
Operatorë krahasimi: = > < >= <= !=
●
AND — të dyja kushtet duhet të jenë të vërteta
●
OR — të paktën njëri kusht duhet të jetë i vërtetë
●
Tekstet shkruhen me thonjëza të vetme ' ' — jo dyfishe
03
ORDER BY — Rendito
Rendit rezultatet sipas kolonës
Fillestare
-- Nga më i vogli tek më i madhi (default) SELECT emri, rroga FROM punonjësit ORDER BY rroga ASC; -- Nga më i madhi tek më i vogli SELECT emri, rroga FROM punonjësit ORDER BY rroga DESC; -- Rendito sipas shumë kolonave ORDER BY departamenti ASC, rroga DESC;
04
LIMIT & DISTINCT
Kufizo numrin e rreshtave dhe largo duplikatet
Fillestare
-- Merr vetëm 10 rreshtat e parë SELECT * FROM punonjësit LIMIT 10; -- Shfaq vetëm vlerat unike SELECT DISTINCT departamenti FROM punonjësit; -- Kombinim SELECT DISTINCT qyteti FROM klientët ORDER BY qyteti LIMIT 5;
05
LIKE & IN — Kërko Pattern
Filtro me pattern teksti dhe lista vlerash
Fillestare
-- LIKE: % = çdo karakter, _ = një karakter SELECT * FROM punonjësit WHERE emri LIKE 'A%'; -- fillon me A WHERE emri LIKE '%ion'; -- mbaron me ion -- IN: si shumë OR bashkë WHERE departamenti IN ('IT', 'HR', 'Finance'); -- NOT IN: ekskludo WHERE qyteti NOT IN ('Prishtinë', 'Prizren');
SQL Mesme
JOIN, GROUP BY, funksione agregimi dhe subqueries.
06
COUNT, SUM, AVG — Funksione Agregimi
Llogarit statistika mbi grupe të dhënash
Mesme
-- Numëro rreshtat SELECT COUNT(*) AS total_punonjës FROM punonjësit; -- Shuma, mesatarja, min, max SELECT SUM(rroga) AS total_rrogave, AVG(rroga) AS rroga_mesatare, MIN(rroga) AS rroga_min, MAX(rroga) AS rroga_max FROM punonjësit;
07
GROUP BY & HAVING
Grupe të dhëna dhe filtro grupe
Mesme
-- Numëro punonjës sipas departamentit SELECT departamenti, COUNT(*) AS numri FROM punonjësit GROUP BY departamenti; -- HAVING filtron GRUPE (jo rreshta) SELECT departamenti, AVG(rroga) AS avg_rroga FROM punonjësit GROUP BY departamenti HAVING AVG(rroga) > 45000 ORDER BY avg_rroga DESC;
⚠
WHERE filtron rreshtat PARA GROUP BY
⚠
HAVING filtron grupe PAS GROUP BY — mund të përdorë funksione agregimi
✓
Rendi i ekzekutimit: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
08
INNER JOIN — Bashko Tabela
Merr rreshtat me lidhje në të dyja tabelat
Mesme
-- Tabela: punonjësit (id, emri, dept_id) -- Tabela: departamentet (id, emri_dept) SELECT p.emri AS punonjësi, d.emri_dept AS departamenti FROM punonjësit p INNER JOIN departamentet d ON p.dept_id = d.id; -- Vetëm punonjësit me departament ekzistues
09
LEFT JOIN, RIGHT JOIN, FULL JOIN
Llojet e ndryshme të bashkimit të tabelave
Mesme
-- LEFT JOIN: të gjithë nga e majta + matching SELECT p.emri, d.emri_dept FROM punonjësit p LEFT JOIN departamentet d ON p.dept_id = d.id; -- Punonjësit pa departament shfaqen me NULL -- Gjej punonjësit BEZ departamenti SELECT p.emri FROM punonjësit p LEFT JOIN departamentet d ON p.dept_id = d.id WHERE d.id IS NULL;
INNER JOIN
Vetëm matching
LEFT JOIN
E gjithë e majta + matching
10
Subqueries — Pyetje brenda Pyetjesh
Përdor rezultatin e një query si input tjetër
Mesme
-- Punonjësit me rroga mbi mesatare SELECT emri, rroga FROM punonjësit WHERE rroga > ( SELECT AVG(rroga) FROM punonjësit ); -- Subquery në FROM (derived table) SELECT dept, avg_rroga FROM ( SELECT departamenti AS dept, AVG(rroga) AS avg_rroga FROM punonjësit GROUP BY departamenti ) sub WHERE avg_rroga > 50000;
SQL Avancuar
Window functions, CTEs, indekse, transaksione dhe optimizim.
11
CTE — Common Table Expressions
Queries të lexueshme dhe rekursive me WITH
Avancuar
-- CTE bazë WITH punonjës_senior AS ( SELECT emri, rroga, departamenti FROM punonjësit WHERE vitet_eksperiencë > 5 ) SELECT departamenti, AVG(rroga) AS avg_senior FROM punonjës_senior GROUP BY departamenti; -- Shumë CTE bashkë WITH total AS (SELECT COUNT(*) n FROM punonjësit), senior AS (SELECT COUNT(*) n FROM punonjësit WHERE vitet > 5) SELECT senior.n * 100.0 / total.n AS pct_senior FROM total, senior;
12
Window Functions — ROW_NUMBER, RANK, LAG
Llogarit mbi grupe pa i shpërbërë
Avancuar
-- Renditja brenda çdo departamenti SELECT emri, departamenti, rroga, RANK() OVER ( PARTITION BY departamenti ORDER BY rroga DESC ) AS renditja FROM punonjësit; -- LAG: vlera e rreshtit të mëparshëm SELECT muaji, shitjet, LAG(shitjet, 1) OVER (ORDER BY muaji) AS muaji_parë, shitjet - LAG(shitjet, 1) OVER (ORDER BY muaji) AS ndryshimi FROM shitjet_mujore;
→
ROW_NUMBER() — numër unik për çdo rresht
→
RANK() — renditje me boshllëqe (1,1,3...)
→
DENSE_RANK() — renditje pa boshllëqe (1,1,2...)
→
LAG/LEAD — vlerat e rreshtave para/pas
→
SUM() OVER() — shumë kumulative
13
Indekset — Shpejto Queries
Optimizo performancën e bazës së të dhënave
Avancuar
-- Krijo indeks bazë CREATE INDEX idx_rroga ON punonjësit(rroga); -- Indeks i kombinuar (shumë kolona) CREATE INDEX idx_dept_rroga ON punonjësit(departamenti, rroga); -- Indeks unik (no duplicates) CREATE UNIQUE INDEX idx_email ON punonjësit(email); -- Shiko planin e ekzekutimit EXPLAIN ANALYZE SELECT * FROM punonjësit WHERE departamenti = 'IT';
14
Transaksionet — ACID
Siguro integritetin e të dhënave
Avancuar
-- Transaksion: transfero para BEGIN TRANSACTION; UPDATE llogaritë SET balanca = balanca - 1000 WHERE id = 1; UPDATE llogaritë SET balanca = balanca + 1000 WHERE id = 2; -- Nëse gjithçka ok COMMIT; -- Nëse ka gabim: anullo gjithçka ROLLBACK;
A
Atomicity — Gjithçka ose asgjë. Nuk ekziston "gjysëm operacioni".
C
Consistency — Baza mbetet në gjendje të vlefshme para dhe pas transaksionit.
I
Isolation — Transaksionet paralele nuk ndërhyjnë njëra-tjetrën.
D
Durability — Pas COMMIT, të dhënat ruhen edhe nëse sistemi rrëzohet.

-- SQL ekzekutohet në këtë rend (jo si shkruhet): 1. FROM -- nga cila tabelë 2. JOIN -- bashko tabelat 3. WHERE -- filtro rreshtat 4. GROUP BY -- grupe 5. HAVING -- filtro grupe 6. SELECT -- zgjidh kolonat 7. DISTINCT -- largo duplikatet 8. ORDER BY -- rendito 9. LIMIT -- kufizon rreshtat
